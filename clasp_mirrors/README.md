# Clasp mirrors (`clasp_mirrors/`)

This tree is generated by **`scripts/clone_clasp_mirrors.mjs`**: one subdirectory per **Apps Script `scriptId`**, cloned with **`clasp clone`** / refreshed with **`clasp pull`**.

**Git:** Only **safe** artifacts belong in the remote repo: `README.md`, **`PROJECT_INDEX.md`** / **`PROJECT_INDEX.tsv`** (Google project **titles** ↔ `scriptId` ↔ folder, from `clasp list`), **`MIGRATION_CHECKLIST.tsv`**, **`MANIFEST.json`**, and **`.clasp.json`** per project. **`*.js`** and **`appsscript.json`** under each mirror are **gitignored** — script files contain secrets; `appsscript.json` is recreated by clasp. After `git clone`, run **`clasp pull`** inside each `clasp_mirrors/<scriptId>/` (or run `node scripts/clone_clasp_mirrors.mjs`) to populate local files.

**Why folders are still long IDs:** [Apps Script](https://script.google.com/home) shows friendly names like *Agroverse - QR code creation web service*; **`clasp`** expects each mirror directory to match the project id. Use **`PROJECT_INDEX.md`** to look up `scriptId` and open the editor (e.g. [`.../projects/1slQVojn5P2wC7l5LdFesFT243afkZ2HQ9no9mciExl574VeOe3Wom2rW/edit`](https://script.google.com/home/projects/1slQVojn5P2wC7l5LdFesFT243afkZ2HQ9no9mciExl574VeOe3Wom2rW/edit)).

**Git vs mirror (`Code.js`, etc.):** **`clasp_mirrors/**/*.js`** and **`clasp_mirrors/**/appsscript.json`** are **gitignored** (secrets, churn). **Pull requests will not list `Code.js` — that is normal.** Reviewers rely on tracked **`google_app_scripts/**/*.gs`** (e.g. **`tdg_inventory_management/web_app.gs`**) as the committed source. Before **`clasp push`**, sync **`Code.js`** to match the **`.gs`** you want to ship (single-file projects often: `cp google_app_scripts/<domain>/web_app.gs clasp_mirrors/<scriptId>/Code.js` from repo root). After **`clasp pull`**, merge or copy into **`google_app_scripts/**`** if the cloud changed.

**`clasp` runs only under** **`clasp_mirrors/<scriptId>/`**, not under **`google_app_scripts/**`**.

## Regenerate

```bash
cd /path/to/tokenomics
node scripts/clone_clasp_mirrors.mjs
```

- **`PROJECT_INDEX.md`** / **`PROJECT_INDEX.tsv`** — sortable map from **Google project name** → `scriptId` → `clasp_mirrors/...` path + editor link.
- **`MIGRATION_CHECKLIST.tsv`** — each row is a `google_app_scripts/**/*.gs` file: basename match, content-hash match against mirrored files, embedding in `Code.js`, or `UNMAPPED`.
- **`MANIFEST.json`** — last run metadata, **`projects`** list `{ title, scriptId }`, and clone/pull errors.

Options: `--dry-run`, `--skip-clone` (checklist only), `--force-clone` (delete `<scriptId>/` and re-clone — **destructive**).

## Workflow

1. **Resolve `scriptId`:** **`PROJECT_INDEX.md`** (Google title → folder path).
2. **Edit what you commit:** Prefer **`google_app_scripts/<area>/*.gs`** so changes show up in **PRs**. Inventory web app: **`web_app.gs`** ↔ mirror **`Code.js`** in **`1QtK-InsHH6SBtxoxc33-y4vQvuNkbhlkUi_9S1X-AaEgIlSlygM1iZtP/`**.
3. **Deploy:** From **`clasp_mirrors/<scriptId>/`**, ensure **`Code.js`** (and siblings) match the **`.gs`** you intend to ship, then **`clasp push`**. After **`clasp pull`** from Google, merge or copy into **`google_app_scripts/**`** if production drifted.
4. Resolve **`UNMAPPED`** / **`BASENAME_ONLY`** rows in **`MIGRATION_CHECKLIST.tsv`** by aligning names or folder mapping (cloud **`Code.js`** vs split **`*.gs`**).

## Security

- **Rotate** any token that ever appeared in a committed `Code.js` (or other mirror `.js`): BotFather Telegram tokens, JWTs, generic high-entropy keys — treat them as compromised if they hit GitHub, even after removal from the latest commit.
- Prefer **Script Properties** / env-style config in Apps Script instead of literals in source.
- Do not commit **`clasp_mirrors/**/*.js`** or **`clasp_mirrors/**/appsscript.json`** (enforced in root `.gitignore`).
